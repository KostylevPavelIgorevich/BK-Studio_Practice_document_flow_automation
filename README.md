Documentob Diplom — программа для электронного документооборота железнодорожных перевозок.

Она позволяет оформлять заявки (ГУ-12, ГУ-27, ГУ-2б, ГУ-36, ЛУ-59) и накладные, автоматически создавать поля ввода из HTML-шаблонов, валидировать данные, сохранять их в базу SQLite и печатать документы в двух стилях (ГУ-12 и ГУ-114). Программа имеет десктоп-обёртку на Tauri (React + Rust) и бэкенд на Laravel.

Архитектура проекта состоит из двух папок: backend (Laravel, API, SQLite, папка public/templates с HTML-шаблонами) и frontend (React, TypeScript, Tauri). В корне проекта лежит PowerShell-скрипт start.ps1, который автоматически устанавливает все зависимости, создаёт файл .env, генерирует ключ приложения, выполняет миграции базы данных, запускает Laravel сервер в отдельном окне и затем открывает Tauri-приложение.

Для запуска на любом компьютере необходимо установить PHP версии 8.1 или новее (добавив его в системную переменную PATH), Composer, Node.js (LTS версию) и Rust. После этого нужно распаковать архив с проектом, открыть PowerShell в корневой папке проекта и один раз выполнить команду: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force. Затем можно запускать скрипт командой .\start.ps1. При первом запуске скрипт самостоятельно установит все зависимости Composer и npm, создаст базу данных, выполнит миграции и сиды. В дальнейшем для запуска достаточно одной команды .\start.ps1.

Перед первым запуском также необходимо проверить настройки PHP. Откройте файл C:\php\php.ini, найдите строки extension=fileinfo, extension=zip, extension=pdo_sqlite, extension=sqlite3. Если они начинаются с точки с запятой, например ;extension=fileinfo, удалите этот символ в начале каждой строки, чтобы раскомментировать расширения. Если каких-то строк нет, просто добавьте их в конец файла в таком виде: extension=fileinfo, extension=zip, extension=pdo_sqlite, extension=sqlite3, каждое на новой строке. После этого сохраните файл.

Приложение автоматически сканирует папку backend/public/templates/ на наличие HTML-файлов. Для каждого файла извлекаются плейсхолдеры вида {{ field_name }}, а подписи полей берутся из соседних элементов с классами field-label, th или label. Все поля становятся обязательными для заполнения, а их типы (дата, число, текст) определяются автоматически. Это позволяет добавлять новые формы простым копированием HTML-файла.

Пользователи делятся на две роли: администратор (управление группами и пользователями) и обычный пользователь (оформление документов). Логин и пароль по умолчанию задаются сидами, например admin/admin и student/student. При заполнении заявки можно перейти к накладной, и тогда данные из заявки подставляются в накладную автоматически. Печать документов доступна в двух стилях, выбор стиля осуществляется через модальное окно.

Программа работает полностью локально, не требует доступа к интернету. База данных — SQLite, файл backend/database/bd.sqlite. Все пути в проекте относительные, поэтому проект можно переносить на любой компьютер без необходимости перенастройки. Для разработки достаточно изменить файлы в папках frontend/src или backend и перезапустить скрипт.

Если возникают ошибки соединения ERR_CONNECTION_REFUSED, это означает, что не запущен бэкенд. Проверьте, открыто ли окно с командой php artisan serve. При проблемах с политикой выполнения PowerShell выполните указанную выше команду Set-ExecutionPolicy. Если система не распознаёт команды php или composer, значит, пути к PHP или Composer не добавлены в переменную PATH — добавьте их и перезапустите терминал. Если возникает ошибка could not find driver (SQLite), значит, в файле php.ini не включены расширения pdo_sqlite и sqlite3 — добавьте их, как описано выше.
